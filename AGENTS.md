## 概览

DizzyPlay 是一个基于 **Tauri v2** 的桌面客户端，为 [dizzylab](https://www.dizzylab.net) 音乐平台提供原生体验。前端使用 **Vue 3 + Element Plus**，后端使用 **Rust**。

### 技术栈

| 层级      | 技术                      | 说明                                                                   |
| --------- | ------------------------- | ---------------------------------------------------------------------- |
| 桌面框架  | Tauri v2                  | 跨平台桌面壳，提供原生窗口、文件系统、进程通信                         |
| 前端框架  | Vue 3 (Composition API)   | `<script setup>` + `ref`/`computed`/`watch`/`reactive`                 |
| UI 库     | Element Plus              | `el-button`, `el-card`, `el-dialog`, `el-progress`, `el-pagination` 等 |
| 图标      | `@element-plus/icons-vue` | 全局注册，模板中直接使用 `<el-icon><Headset /></el-icon>`              |
| 路由      | vue-router 4              | 懒加载路由，`keep-alive` 缓存 HomePage 和 DiscList                     |
| 状态管理  | 无 Pinia/Vuex             | 使用 `reactive` + 模块级单例 + 自定义事件                              |
| 构建      | Vite 6                    | 开发服务器代理 `/apis/` `/cdn/` `/streaming/` 到 dizzylab              |
| 包管理    | pnpm                      |                                                                        |
| Rust 后端 | reqwest + tokio           | HTTP 代理、文件缓存、MP3 时长解析、流式下载                            |
| 序列化    | serde + serde_json        | Rust ↔ JS 数据交换                                                     |

## 架构模式与关键约定

### 1. 双环境运行

应用可在 **Tauri 桌面环境** 和 **浏览器开发环境** 下运行。代码中通过 `isTauri` 检测：

```javascript
const isTauri =
  typeof window !== "undefined" &&
  window.__TAURI_INTERNALS__ &&
  typeof window.__TAURI_INTERNALS__.invoke === "function";
```

- **Tauri 环境**：通过 `window.__TAURI_INTERNALS__.invoke("command_name", args)` 调用 Rust 后端
- **浏览器环境**：通过 Vite proxy（`/apis/` → `https://www.dizzylab.net/apis/`）直接 fetch

### 2. 前端 → Rust 通信

所有 Tauri 命令在 [`src-tauri/src/lib.rs`](src-tauri/src/lib.rs:25) 的 `invoke_handler` 中注册。

**调用方式**：

```javascript
// 前端调用 Rust 命令
const result = await window.__TAURI_INTERNALS__.invoke("command_name", {
  param1: "value1",
  param2: "value2",
});
```

**Rust 端定义**：

```rust
#[tauri::command]
pub async fn command_name(param1: String, param2: String) -> Result<String, String> {
    // ...
}
```

### 3. 缓存系统

**缓存架构**：

| 缓存类型      | 存储位置        | 用途                         | 命令                                                             |
| ------------- | --------------- | ---------------------------- | ---------------------------------------------------------------- |
| JSON 数据缓存 | `cache/datas/`  | API 响应、专辑详情、列表数据 | `save_cache` / `load_cache` / `delete_cache` / `clear_cache_dir` |
| 图片缓存      | `cache/images/` | 封面、头像                   | `save_image_cache` / `load_image_cache`                          |
| 音乐缓存      | `cache/music/`  | 流式音频文件                 | `save_music_cache` / `load_music_cache`                          |

**前端缓存 API**（在 [`src/services/api.js`](src/services/api.js:81) 中封装）：

```javascript
import { saveCache, loadCache, clearCache } from "../services/api.js";

// 保存
await saveCache("homepage_discs", data);

// 加载（maxAgeSecs 默认 300 = 5 分钟）
const data = await loadCache("homepage_discs", 300);

// 清除
await clearCache("homepage_discs");
```

**缓存刷新机制**：全局 `app-refresh` 自定义事件触发时，各页面清除缓存并重新加载数据。

### 4. 全局事件通信

应用大量使用 `window.dispatchEvent(new CustomEvent(...))` 进行跨组件通信：

| 事件名              | 触发者                 | 监听者                          | 用途               |
| ------------------- | ---------------------- | ------------------------------- | ------------------ |
| `add-to-playlist`   | AlbumDetail, PlayerBar | PlayerBar                       | 添加曲目到播放列表 |
| `add-download-task` | AlbumDetail            | DownloadManager                 | 添加下载任务       |
| `app-refresh`       | TopNavbar              | HomePage, DiscList, AlbumDetail | 全局刷新           |
| `settings-changed`  | SettingsPage           | AlbumDetail, PlayerBar          | 设置变更通知       |

### 5. 下载系统架构

下载系统涉及前端、Rust 后端、事件通信三层：

```text
AlbumDetail.vue
  └─ downloadManager.addTask(url, label, discId, albumTitle)
       └─ DownloadManager (全局单例)
            ├─ Tauri 环境: invoke("download_file", { url, save_path, csrf_token, session_id, task_id, offset })
            │    └─ Rust download.rs
            │         ├─ 流式下载 (reqwest bytes_stream)
            │         ├─ 进度推送: app_handle.emit("download-progress", { downloaded, total, percent })
            │         ├─ 暂停: 检查 CANCEL_TOKENS[task_id] → 返回 "下载已暂停"
            │         └─ 续传: Range: bytes={offset}- 头 + 文件追加模式
            └─ 浏览器环境: fetch + ReadableStream (chunked)
                 └─ 通过 Vite proxy 下载
```

**关键文件**：

- [`src/services/downloadManager.js`](src/services/downloadManager.js:23) — 全局下载管理器
- [`src-tauri/src/modules/download.rs`](src-tauri/src/modules/download.rs:1) — Rust 下载引擎
- [`src/views/DownloadPage.vue`](src/views/DownloadPage.vue:1) — 下载管理 UI

### 6. 分页与封面懒加载

列表页（HomePage, DiscList）使用 **前端分页** + **封面懒缓存**：

```javascript
const pageSize = 20;
const currentPage = ref(1);

const paginatedDiscs = computed(() => {
  const start = (currentPage.value - 1) * pageSize;
  return discs.value.slice(start, start + pageSize);
});

// 仅缓存当前分页可见的封面
watch(
  paginatedDiscs,
  (newItems) => {
    if (newItems && newItems.length > 0) {
      cacheVisibleCovers(newItems);
    }
  },
  { immediate: false },
);
```

### 7. keep-alive 缓存策略

在 [`src/App.vue`](src/App.vue:29) 中：

```html
<keep-alive :include="['HomePage', 'DiscList']">
  <component :is="Component" />
</keep-alive>
```

- **HomePage** 和 **DiscList** 被 `keep-alive` 缓存，切换回时不重新加载
- 其他页面（AlbumDetail, UserDetail 等）不缓存，每次进入重新挂载
- 路由参数变化时通过 `watch(() => route.params.id, ...)` 重新加载数据

### 8. 用户页面 HTML 解析

[`src/services/htmlParser.js`](src/services/htmlParser.js:99) 中的 `parseUserPageDiscs()` 用于解析 dizzylab 用户页面的 HTML：

- **步骤 1**：提取 `tab-pane#album` 内容区域（排除导航栏、页脚）
- **步骤 2**：匹配 `card-group > card > card-body + card-footer` DOM 结构提取唱片信息
- **步骤 3**：回退匹配社团条目（`/l/{name}` 结构）

## 常见开发模式

### 添加新的 Tauri 命令

1. 在对应的 Rust 模块中实现 `#[tauri::command]` 函数
2. 在 [`src-tauri/src/lib.rs`](src-tauri/src/lib.rs:25) 的 `invoke_handler` 中注册
3. 在 [`src-tauri/capabilities/default.json`](src-tauri/capabilities/default.json:8) 中添加权限（如果需要）
4. 在前端通过 `window.__TAURI_INTERNALS__.invoke("command_name", args)` 调用
5. 在 [`src/services/api.js`](src/services/api.js:33) 中封装为可导出函数（推荐）

1. 在 [`src/views/`](src/views/) 下创建 `.vue` 文件
2. 在 [`src/router.js`](src/router.js:3) 中添加路由（懒加载）
3. 在 [`src/components/SidebarNav.vue`](src/components/SidebarNav.vue:20) 的 `menuItems` 中添加导航项

### 添加 API 端点

1. 在 [`src/services/api.js`](src/services/api.js:545) 中使用 `apiGet()` 或 `apiPost()` 封装
2. 如果需要 Rust 代理，在 [`src-tauri/src/modules/api.rs`](src-tauri/src/modules/api.rs:26) 中已有通用 `proxy_api_get` 和 `proxy_api_post`

## 关键注意事项

### 1. Tauri v2 特有 API

- **事件监听**：使用 `import { listen } from "@tauri-apps/api/event"`，**不是** `window.__TAURI_INTERNALS__.listen`
- **窗口操作**：使用 `import { getCurrentWindow } from "@tauri-apps/api/window"`
- **剪贴板**：使用 `import { writeText } from "@tauri-apps/plugin-clipboard-manager"`
- **打开链接**：`window.__TAURI_INTERNALS__.invoke("plugin:opener|open_url", { url })`

### 2. Rust 后端注意事项

- 使用 `std::sync::LazyLock`（Rust 1.80+）替代 `once_cell` 做惰性静态初始化
- 异步命令必须用 `async fn` + `pub async fn`
- 错误处理统一返回 `Result<T, String>`，错误信息用中文
- 下载进度通过 `app_handle.emit("event-name", payload)` 推送（需 `use tauri::Emitter;`）
- 全局状态（如下载取消标志）使用 `static CANCEL_TOKENS: LazyLock<Mutex<HashMap<String, bool>>>`

### 3. 前端注意事项

- **不要**在组件卸载时销毁 DownloadManager（它是全局单例，生命周期 = 应用生命周期）
- **不要**在非 Tauri 环境调用 `invoke`（先检查 `isTauri`）
- 列表页的 `watch(() => route.path, ...)` 用于处理路由变化，但 `keep-alive` 缓存的组件不会重新 `onMounted`
- UserDetail 组件复用（相同路由参数）时，`onMounted` 不触发，需通过缓存恢复状态

### 4. 构建与运行

```bash
# 开发（浏览器模式，仅前端）
pnpm dev

# 开发（Tauri 桌面模式）
pnpm tauri dev

# 生产构建
pnpm tauri build --no-bundle
```

## 数据流示例：下载流程

```text
用户点击"下载"按钮
  │
  ▼
AlbumDetail.vue: startDownload(index)
  │  downloadManager.addTask(url, label, discId, albumTitle)
  ▼
DownloadManager.addTask()
  │  1. 创建任务对象 { id, url, label, discId, albumTitle, status: PENDING, ... }
  │  2. 保存到 localStorage
  │  3. 自动调用 startTask(taskId)
  ▼
DownloadManager.startTask(taskId)
  │  1. 获取下载路径 (loadUserConfig("downloadPath"))
  │  2. 获取 Cookie 凭证 (csrfToken + sessionId)
  │  3. 调用 _tauriDownload() 或 _browserDownload()
  ▼
_tauriDownload(task, csrfToken, sessionId)
  │  1. listen("download-progress") 注册进度监听
  │  2. invoke("download_file", { url, save_path, csrf_token, session_id, task_id, offset })
  ▼
Rust: download_file()
  │  1. 注册取消标志 CANCEL_TOKENS[task_id] = false
  │  2. 如果 offset > 0: Range: bytes={offset}- 头 + 文件追加模式
  │  3. reqwest.get(url).send() → bytes_stream()
  │  4. 逐块写入文件，每块检查取消标志
  │  5. 每 1% 进度变化 emit("download-progress", { downloaded, total, percent })
  │  6. 完成 → 清理取消标志
  │  7. 取消 → 返回 "下载已暂停" 错误
  ▼
DownloadManager 收到进度事件
  │  更新 task.downloadedBytes, task.totalBytes, task.percent
  │  _notify() → DownloadPage.vue 更新 UI
  ▼
DownloadPage.vue
  │  <el-progress :percentage="task.percent" />
  │  {{ formatSize(task.downloadedBytes) }} / {{ formatSize(task.totalBytes) }}
```

## 代码风格约定

- **Vue 组件**：使用 `<script setup>` Composition API
- **命名**：驼峰命名（camelCase），组件名 PascalCase
- **CSS**：Scoped styles，使用 CSS 变量（`var(--el-color-primary)` 等）
- **Rust**：遵循 Rust 2018 edition 风格，错误消息用中文
- **注释**：关键逻辑和复杂函数必须写中文注释
- **导入顺序**：Vue → 组件 → 服务 → 工具
